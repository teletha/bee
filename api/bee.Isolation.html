Isolation<V>

  • V

    The type of the value returned by the #bridge() method. Use Void if no return value is needed (and override #isolate() instead).

  • Serializable

Provides a mechanism to execute code within an isolated context defined by a specific set of library dependencies.

This class achieves isolation by creating a dedicated PriorityClassLoader loaded with the specified dependencies and their transitive runtime dependencies. It then uses a serialization-deserialization trick within its constructor to switch the execution context to this isolated classloader for the #isolate() or #bridge() method.

Subclasses should override either #isolate() for procedures that do not return a value, or #bridge() for functions that return a value of type V. The constructor automatically detects which method is overridden (preferring isolate() if both are somehow overridden) and executes it within the isolated context.

Use the #load(Class, String) and #create(Class, String) methods within your overridden isolate() or bridge() method to interact with classes loaded from the specified dependencies.

Isolation(Stringdependencies)

Stringdependencies

A list of dependency descriptors (e.g., "groupId:artifactId:version") required for the isolated execution context.

Sets up the dependency context and executes the isolated code. All operations involving dependency classes (loading, instantiation, method calls) should occur within the overridden isolate() or bridge() method to ensure they happen in the isolated context.

isolate()void

This method is intended to be overridden by subclasses to perform actions within the isolated classloader context that do not produce a return value.

The code inside this method will execute with the dependencies specified in the constructor loaded preferentially by the isolated classloader. Use #load(Class, String) and #create(Class, String) to interact with classes from these dependencies.

If this method is overridden, the #bridge() method will typically not be executed.

The default implementation does nothing.

bridge()V

V

The computed result of type V.

This method is intended to be overridden by subclasses to perform actions within the isolated classloader context that produce a return value of type V.

The code inside this method will execute with the dependencies specified in the constructor loaded preferentially by the isolated classloader. Use #load(Class, String) and #create(Class, String) to interact with classes from these dependencies.

If the #isolate() method is overridden by the subclass, this bridge() method will typically not be executed. The result of executing this method is stored and can be retrieved using #result().

The default implementation returns null.

load(ClassTtype, Stringfqcn)ClassT

T

The expected type of the class.
ClassTtype

A Class object representing the expected type T, used only for compile-time type checking and casting convenience of the return value.
Stringfqcn

The fully qualified class name (e.g., "com.example.MyClass").
ClassT

The loaded Class object, cast to Class<T>.

Loads a class by its fully qualified name using the internal PriorityClassLoader associated with this dependency context. This ensures the class is loaded from the specified dependencies if available, respecting the classloader's priority rules.

create(ClassTtype, Stringfqcn)T

T

The expected type of the instance.
ClassTtype

A Class object representing the expected type T. This is used both for loading the correct class via #load(Class, String) and for type safety of the returned instance.
Stringfqcn

The fully qualified class name of the class to load and instantiate.
T

A new instance of the loaded class, cast to T.

Loads a class by its fully qualified name using the internal PriorityClassLoader and then creates a new instance of it using I#make(Class) (Kiss DI). This is useful for instantiating classes from the dependency context, potentially utilizing dependency injection if configured via kiss.I.

result()V

V

The result from the #bridge() execution, or null.

Returns the result computed and returned by the #bridge() method, if it was executed. If the #isolate() method was executed instead (because it was overridden), this method will return the initial value of the result field (typically null).

with(Stringdependencies)IsolationVoid

Stringdependencies

IsolationVoid

A new Isolation instance whose isolate() method does nothing. The primary effect is the setup of the class loader with the dependencies during the construction of this object.

Factory method to create an Isolation instance configured with the specified dependencies, designed for cases where no return value is needed (i.e., side effects only). The returned instance overrides #isolate() with an empty implementation. A subclass would typically be created to override isolate with actual logic. This factory might be less useful now compared to directly creating a subclass that overrides isolate.
