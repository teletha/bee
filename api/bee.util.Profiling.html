Profiling

  • AutoCloseable

A simple built-in profiler for measuring the execution time of code blocks.

This profiler uses a static collection to store timing records globally. It supports nested profiling scopes using a try-with-resources pattern. The time reported for a scope represents the "self-time" – the time spent directly within that scope, excluding time spent in nested (child) scopes.

Usage Example: 


 // Start profiling the "Data Processing" task
 try (Profiling ignored = Profiling.of("Data Processing")) {
     // Code for data processing...

 // Start profiling a sub-task &amp;#34;Validation&amp;#34;
 try (Profiling ignored2 = Profiling.of(&amp;#34;Validation&amp;#34;)) {
     // Code for validation...
     Thread.sleep(50); // Simulate work

} // &quot;Validation&quot; scope ends here
// More data processing code...
Thread.sleep(100); // Simulate work
} // &quot;Data Processing&quot; scope ends here
// Later, display the results
Profiling.show(ui); // Assuming 'ui' is a UserInterface instance

In the example above, the time reported for "Data Processing" will be approximately 100ms, as the 50ms spent in "Validation" is excluded from its parent's self-time. The time for "Validation" will be approximately 50ms.

Note: This profiler is designed for single-threaded usage scenarios or scenarios where nesting occurs strictly within the same thread. The global state might lead to incorrect nesting if Profiling.of() and close() calls are interleaved across different threads without proper external synchronization. The collection of results (`records`) is thread-safe.

close()void

Stops the timer for the current scope and restores the previous scope as the active one. This method is automatically called when using try-with-resources. When the previous scope is restored, its timer is resumed.

measureJVMStartup()void

Calculates the approximate JVM startup time until this method is called. It creates a special &quot;JVM startup&quot; profiling record. This relies on ProcessHandle#info() which might not be available or accurate on all platforms/JVMs.

This method is somewhat special as it tries to measure time spent even before the application's main logic (and potentially this profiler) is fully operational.

of(Stringname)Profiling

Stringname

The name of the scope to profile.
Profiling

An AutoCloseable instance representing the new scope. Its #close() method must be called (typically via try-with-resources) to stop the timer and restore the previous scope.

Starts a new profiling scope with the given name.

This method should be used with try-with-resources: 

try (Profiling p = Profiling.of("My Task")) { ... }

When a new scope is started:

  1. The currently active scope (if any) is paused.
  2. A new Profiling instance is created and added to the global records.
  3. The new instance becomes the current active scope.
  4. The timer for the new scope is started.

show(UserInterfaceui)void

UserInterfaceui

The UserInterface used for outputting the results.

Aggregates the collected profiling data, formats it, and displays the top results (longest durations first) using the provided UserInterface.

Only durations of 100ms or more are included. Up to the top 10 entries are shown. The output format includes the duration (e.g., "1.234s") and the scope name.
