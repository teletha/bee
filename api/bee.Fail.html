Fail

  • Exception

Signals a recoverable failure during the build process.

This exception is thrown when a build-related operation fails in a way that should be reported to the user, such as misconfiguration, missing dependencies, or unsupported environments.

Unlike critical runtime errors, this exception is unchecked (extends RuntimeException) and intended to be handled by the build infrastructure or surfaced to the user with actionable context.

Each Fail instance includes:

  • A mandatory reason message that describes what went wrong.
  • An optional list of suggested solutions that help resolve the issue.
  • An optional underlying cause (wrapped Throwable).

Example: 


 throw new Fail("Missing JDK installation")
     .solve("Install JDK 21 or later")
     .solve("Ensure JAVA_HOME is set correctly")
     .reason(e);

Fail(Stringreason, ListStringsolutions)

Stringreason

The detail message explaining the reason for the failure. Must not be null.
ListStringsolutions

A list of suggested solutions. May be null or empty.

Constructs a new Fail exception with the specified reason message and an initial list of solutions. If the provided list of solutions is null, an empty list is used internally. The provided list is copied, so subsequent modifications to the original list will not affect this instance.
